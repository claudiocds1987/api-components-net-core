

ARQUITECTURA: Patrón Repositorio y capas

Carpeta "Controllers" solo manejan solicitudes HTTP y devuelven respuestas.

Carpeta "Services" manejan las reglas de negocio.

Carpeta "Repositories" manejan el acceso a la base de datos.
Nota: Las validaciones se hacen por ejemplo para Employee en EmployeeService.cs

Carpeta "Persistence/Configurations" manejan el mapeo de EF Core.

Este proyecto utiliza una Arquitectura en Capas que implementa el Patrón Repositorio. 
La meta es la Separación de Responsabilidades para que cada parte del código tenga un único trabajo.

* Controllers (Capa de Presentación):

	- Función: Solo manejan el tráfico HTTP.
	- Acciones: Reciben la solicitud, llaman al Servicio correspondiente y devuelven la respuesta web 
	  (200 Ok, 404 Not Found, etc.).
	- Depende de: IEmployeeService (la interfaz del servicio).

* Services (Capa de Aplicación/Negocio):

	- Función: Contienen la Lógica de Negocio Central.
	- Acciones: Realizan validaciones funcionales (ej. "El empleado debe ser mayor de 18 años"). 
	  Orquestan llamadas a uno o más Repositorios.
	- Depende de: IEmployeeRepository (la interfaz del repositorio).

	Nota Importante: La lógica de validación de negocio se escribe en el archivo de implementación, por ejemplo, 
	EmployeeService.cs.


* Repositories (Capa de Persistencia):

	- Función: Aísla el acceso a la base de datos.
	- Acciones: Traducen las peticiones del Servicio a comandos de la base de datos (queries IQueryable, Where, FindAsync)
	  usando AppDbContext.
	- Depende de: AppDbContext (Entity Framework Core).

* Persistence/Configurations (Manejan el mapeo avanzado de EF Core "Fluent API"):

	- Función: Manejan el mapeo y la configuración de EF Core.
	- Acciones: Definen las tablas, relaciones, claves primarias y restricciones de unicidad 
	 (IsUnique,HasIndex, HasMany, etc.), manteniendo el archivo AppDbContext limpio.

Principio Clave: Las capas superiores (Controller, Service) solo ven las Interfaces (I...) de las capas inferiores. 
Esto garantiza que el código de negocio no esté atado a la tecnología de base de datos.

--------------------------------------------------------------------------------------------------------------------------------------

DbSet: Convierte tu clase de C# (Employee) en una tabla de la base de datos 
y te da acceso a las herramientas para manipular los datos de esa tabla.
Ej: public DbSet<Employee> Employees { get; set; }
